# リクエストメール導線ハンドオフ

作成日: 2026-07-07

## 目的
`さがす` タブから届く「こんなストレッチが欲しい」を、β配布中の改善材料として安全に受ける。

## 現在のアプリ側
- 送信先: `kyou-no@ogatore.jp`
- 起点: `さがす` タブのリクエストボタン
- 送信方式: `mailto:`
- 件名: `ストレッチのリクエスト（きょうのオガトレ）`
- 本文テンプレート:
  - 検索語または選択タグ
  - `きょうのオガトレ「さがす」から送信`

## β配布前に必要な設定（2026-07-15更新: 本人確認のうえ決定・設定済み）
- `kyou-no@ogatore.jp` のエイリアスまたはグループを作成 → 設定済み
- 受信先を決める → 決定済み
- Gmailラベルを作成: `きょうのオガトレ/リクエスト` → 設定済み
- 受信トレイを汚さない運用にするか決める → Gmailフィルタで運用開始
- 朝ブリーフへ件数だけ載せるか決める → 「朝6時ブリーフにリクエスト集計を追加する」が発動済み（ハブ=alan本体側の作業のためこのリポジトリでは実装しない）

## 推奨Gmailフィルタ
条件:
- To: `kyou-no@ogatore.jp`
- Subject: `ストレッチのリクエスト（きょうのオガトレ）`

処理:
- ラベル付与: `きょうのオガトレ/リクエスト`
- 既読化はしない
- 重要マークは付けない
- 受信トレイスキップは、β初期は保留。見逃しが不安なら受信トレイに残す

## 朝ブリーフに載せるなら
毎朝確認する値:
- 前日0:00以降の新着件数
- 多い検索語/本文キーワード
- 明らかな不具合報告
- 返信が必要そうなもの

出力例:
`#きょうのオガトレ リクエスト: 昨日3件。多い要望は「肩こり」「寝る前」。不具合報告なし。`
