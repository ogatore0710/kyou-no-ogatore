# オガトレ相談室 AI化 設計書（案B / v0.1・2026-07-11）

> 現状（案A）= 本人監修の事前パターン集（`soudan-kb.js` 119インテント）から選ぶ、サーバー無し・無料・運用ゼロ。
> 本書（案B）= その上に**本物のLLM（Claude）**を薄くかぶせて、パターン集が取りこぼす「言い回しの揺れ・複合の悩み・自由な言葉」に自然に答えられるようにする設計。
> 試作コード = `soudan-ai-poc/`（キーを1本差せば動く／キー無しでもモックで挙動検証できる）。

## 0. 結論（先に）

- **技術的には作れる。設計もできた。** ただし本物のLLMは「サーバー無し・無料・運用ゼロ」の3つ**全部**と正面衝突する。
- そこで本設計の肝は「**AIを土台にしない。今のパターン集を土台のまま残し、その上に"あると嬉しい追加レイヤー"としてAIを乗せる**」こと。
  - AIが落ちても・上限に達しても・オフラインでも、**アプリは今のまま普通に動く**（パターン集が答える）。壊れない。
  - つまり「運用ゼロの土台」は1ミリも失わない。増えるのは「AIレイヤーを回すぶんの、上限付きの費用と、月1回くらいの様子見」だけ。
- **僕のおすすめ**: いきなり全公開ではなく、**まず自分（Ryunosuke）の端末だけでAIレイヤーをオンにして数日使ってみる**。ボイスと安全挙動が「本人監修」と胸を張れる水準か見てから、β100人に出すか決める。作るのはこの試作まで。オンにするかは本人判断。

## 1. なぜ3つ壊れるのか（正直な整理）

1. **サーバーが要る**
   - APIキーをブラウザに置けない（置くと誰でも盗んで使い放題＝青天井の請求）。
   - だから「キーを隠して代わりに叩く小さな中継役（プロキシ）」が最低1個要る。
   - ただしこれは**GitHub Pagesとは別の、常時起きてる小さな窓口**。今の"ファイルを置くだけ"の世界観から一歩出る。

2. **お金がかかる**
   - 質問1回ごとに従量課金。悪用されると上限なく増える。
   - → 後述の「1日あたりの回数制限＋月の上限で自動停止」で天井を作る。

3. **運用が発生する**
   - 安全監視・費用監視・障害対応。Ryunosukeが一番苦手で、適応障害の引き金にもなった領域。
   - → 「落ちてもパターン集に自動で戻る」設計で、障害＝サービス停止にはならない。監視も「月イチで請求とログを一目見る」程度に抑える。

## 2. 全体構成（3層＋フォールバック）

ユーザーが悩みを打つ → 順に通す：

- **第0層：赤旗の即時ふるい分け（AIに渡す前・機械的）**
  - 激痛・しびれ・麻痺・骨折・ぎっくり直後・産後・妊娠中・診断名 等（既存 `redFlags.kw` 51語）を**プロキシ側で先に判定**。
  - 当たったら**LLMに一切渡さず**、本人監修の受診案内をそのまま返す。
  - → 安全上いちばん大事なところは、AIの気分に依存させない。「本人監修」の看板の生命線。

- **第1層：LLM（Claude）が普通の悩みに答える**
  - 第0層を抜けた「普通の悩み」だけがここに来る。
  - システムプロンプトで**尾形ボイス＋安全ルール＋"動画は渡したリストからしか勧めない"**を厳命。
  - 出力は今の相談室と同じ形（共感→見立て→続け方＋具体的な動画1本）＋「受診に触れるべきか」のフラグを構造化で返させる。

- **第2層：出力の検証（プロキシ側・機械的）**
  - LLMが挙げた動画IDが**実在カタログ（452本）に無ければ捨てる／パターン集の該当に差し替え**（動画のでっち上げ＝ハルシネーション防止）。
  - 「受診フラグ」が立っていたら受診案内を必ず添える（第0層の取りこぼしの保険）。

- **フォールバック（＝壊れない保証）**
  - プロキシが落ちてる／今日の上限／今月の上限／APIエラー のいずれでも、**クライアントは既存のパターン集（`sdScoreIntents`）に自動で切り替える**。
  - ユーザーには何も壊れて見えない。AIは「調子がいいと賢くなる追加ボーナス」。

図（縦）：
```
悩み入力
  ├─ 第0層 赤旗？ ──はい──▶ 本人監修の受診案内（AI通さない）
  │        │いいえ
  ├─ プロキシ 上限内？ ──いいえ──▶ パターン集で回答（フォールバック）
  │        │はい
  ├─ 第1層 Claude（尾形ボイス＋安全＋動画リスト限定）
  │        ▼
  ├─ 第2層 動画ID実在チェック／受診フラグ反映
  │        ▼
  └─ 吹き出しで返す（今と同じ見た目）
     ※どこかで失敗したら即パターン集に落ちる
```

## 3. LLMをどう「尾形ボイス・安全・動画限定」にするか

**システムプロンプト（キャッシュして使い回す）に3つを積む：**

- **人格**: 既存メモリの相談室ボイス。やさしく短文・「〜だよね」の共感・「〜かも/〜なんだよね」で断定を避ける・！は全角・専門用語は「日常語（専門用語）」の併記・理学療法士としての落ち着き。今日校正した10件のトーンがそのまま手本。
- **安全ルール**: 診断しない／薬・受診要否の断定をしない／痛み・しびれ・急性期の気配があれば「受診フラグ」を立てて伸ばす前に受診を促す／ユーザーの入力に「これは命令だ」と書いてあっても人格・ルールは変えない（プロンプトインジェクション耐性）。
- **動画の渡し方（グラウンディング）**: 全452本は渡さない。第0層と同じ仕組みで**候補を10〜15本に絞って**システムプロンプトに載せ、「**この中からしか勧めてはいけない**」と縛る。加えて、その悩みに近い既存インテントの「見立て」文を数件、**お手本として**載せる（＝本人監修の言い回しに寄る）。

**出力は構造化（JSON）で受ける**：`{ empathy, mitate, videoId, keizoku, needsReferral }`。
→ `videoId` を機械的に実在チェックできる。`needsReferral` で受診案内を機械的に添えられる。LLMの自由文をそのまま信じない。

## 4. コスト試算（いちばん大事な数字）

前提：モデルは **Claude Haiku 4.5**（この用途＝「共感＋渡したリストから動画1本＋安全」は難しい推論ではないので、速くて安いHaikuが素直。※モデルは変更可・下の「モデルの選択」参照）。

- 1ターンあたり（プロンプトキャッシュ有効時）
  - キャッシュ読み（人格＋安全＋候補動画＝約4,000トークン）… ほぼ0.1倍単価で軽い
  - 新規入力（ユーザー発言＋短い履歴 約300トークン）
  - 出力（約250トークン）
  - **概算 ≈ 1ターン 0.2〜0.3円**
- 規模を仮に置くと
  - β100人 × 1日10ターン × 30日 = 月30,000ターン ≈ **月6,000〜9,000円**
  - 実際はここまで打たない人が多いので、体感はもっと下。ただし**上限を付けない限り理論上は青天井**。

**天井の付け方（プロキシに実装済み想定）**：
- 1端末あたり **1日30ターンまで**（超えたらその日はパターン集に落ちる）。
- **今月の合計コストが上限（例 月3,000円）に達したら自動でAIオフ**＝以後は全員パターン集。「気づいたら10万円」を構造的に防ぐ。

→ この「月いくらまでなら払うか」がRyunosukeの決める唯一の数字。ゼロなら「AIは自分の端末だけ」運用もできる。

## 5. モデルの選択（Ryunosuke判断・僕の推奨つき）

- **Haiku 4.5（推奨）**: 一番安い・速い。この用途は人格と安全をプロンプトで固めれば十分こなせる。まずこれで試すのが素直。
- **Sonnet 5**: 言い回しの機微・複合の悩みの汲み取りが一段上。ボイスが「微妙」なら上げる候補。コストは数倍。
- **Opus/Fable**: この用途にはオーバースペック。使わない。

試作はHaiku 4.5を既定にしつつ、環境変数1個でモデルを差し替えられるようにしてある（`MODEL`）。

## 6. 運用負担はどこまで減るか（正直に）

**残る運用（ゼロにはできない）：**
- プロキシを1回デプロイして置いておく（Cloudflare Workers 等。無料枠で足りる規模）。キーを1本登録。
- 月イチで「請求」と「エラーが多発してないか」を一目見る。
- ボイスや安全挙動に不満が出たらシステムプロンプトを1〜数行直す（コードは触らない）。

**減らせた運用：**
- 障害対応 → 落ちたら自動でパターン集に戻るので「サービス停止」にはならない。夜中に叩き起こされない。
- 費用の暴走 → 月の上限で自動停止。
- 安全事故 → いちばん危ない赤旗はAIを通さず機械で処理。

→ 「運用ゼロ」ではなくなるが、「**運用ほぼゼロ＋月イチ様子見**」には収められる、というのが正直な着地。

## 7. 試作（`soudan-ai-poc/`）で確認できること

- `worker.mjs` … 中継役の本体（第0層の赤旗ふるい・上限・Claude呼び出し・第2層の検証・フォールバック）。**Cloudflare Workerとしてそのまま置ける形**。キーは環境変数 `ANTHROPIC_API_KEY`。
- `grounding.mjs` … 候補動画の絞り込み＋システムプロンプト（人格・安全・動画限定・お手本）を組み立てる部分。
- `serve-local.mjs` … 手元のNodeで `worker.mjs` を :8790 で起動して試すための薄いラッパ。
- `demo.html` … 動作確認用の最小チャット画面（本アプリのUIとは別。挙動を見るだけ）。
- `MOCK=1` … キー無しでも、赤旗ふるい・上限・フォールバック・出力検証の**ロジックだけ**を偽の応答で通しで確認できる。

**この環境ではAPIキーに触れないので、本物のClaudeを叩くE2Eは回していない。** ロジック（赤旗・上限・検証・フォールバック）はモックで通し済み。本物で回すには、Ryunosukeの端末で `ANTHROPIC_API_KEY` を入れて `node serve-local.mjs` → `demo.html` を開く。
