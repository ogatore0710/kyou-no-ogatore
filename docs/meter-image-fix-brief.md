# とどくメーター画像 修正指示書（GPT/画像生成AI向け）

対象ファイル: `assets/check/meter.jpg`（アプリの「かたさチェック」内、前屈のお手本として表示）

## 今の問題

前屈している人物の横向き写真に、体の部位を示す点線と文字ラベルが5つ付いている。
上から「ひざ」「すね」「足首」「つま先」「ゆか」の順。

このうち下3つ（足首・つま先・ゆか）は、体の構造上どうしても縦の間隔が狭い部分
（足首〜つま先〜床）を指しているため、ラベルの点線と文字が縦に詰まりすぎて
ほぼくっついて見える。アプリ内では実寸 250px 幅程度の小さな表示になるため、
この詰まりが「文字が被っている」ように見えてしまう。

## やってほしいこと

元の（ラベルを付ける前の）人物写真データから、ラベルの配置だけを作り直してほしい。
写真そのもの（人物・ポーズ・服装・背景）は変更しない。

### ラベルの仕様
- 表示する文字とその指す部位（この5つ、順番も維持）:
  1. ひざ
  2. すね
  3. 足首
  4. つま先
  5. ゆか
- スタイル: ベージュ系の破線（現行踏襲）＋シンプルな明朝体/ゴシック体の黒〜濃灰文字
- 各ラベルは「対象部位から水平に伸びる破線」→「文字」という現行のデザイン言語を維持

### 具体的な直し方（優先度順）
1. **下3つのラベル（足首・つま先・ゆか）の文字を横方向にずらして配置する**
   縦位置は部位の実際の高さに合わせつつ、文字ラベルの横位置を1段ずつ右にずらす
   （階段状にする）ことで、文字同士が重ならないようにする。破線の長さを部位ごとに
   変えて対応してもよい。
2. それでも詰まる場合は、足首・つま先・ゆかの3つだけ文字サイズをひと回り小さくする。
3. 上2つ（ひざ・すね）は現行のままで間隔に問題なし。触らなくてよい。

### やってはいけないこと
- 人物の写真自体の加工・生成（実写を保つ。イラスト化・別人化はNG）
- ラベルの部位・文字・順番を変えること
- 全体の縦横比・构図を変えること（アプリ側で 2:3 比率を前提に表示している）

## 出力
- 元画像と同じアスペクト比（900×1350、縦長2:3）のJPEGまたはPNG
- 完成後、`assets/check/meter.jpg` を差し替える
