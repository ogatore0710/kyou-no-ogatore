# とどくメーター画像 修正指示書（GPT/画像生成AI向け）v2

対象ファイル: `assets/check/meter.jpg`（アプリの「かたさチェック」内、前屈のお手本として表示）

v1の指示でGPTに1回生成させたが、まだ以下の問題が残っている。この指示書はその修正版。

## 前回生成した画像で残っている問題

1. **文字が小さくて読みにくい**
   アプリ内では実寸 250px 幅程度の小さな表示になるため、現状の文字サイズでは
   小さすぎて読み取りづらい。フォントは「Mプラス（M PLUS 1p、太め・丸みのある
   ゴシック体）」相当のものを使い、サイズもひと回り大きくすること。
2. **「ひざ」の線の位置が高すぎる**
   実際の膝関節（脚が曲がっている位置）よりも上、太もも寄りを指してしまっている。
   もう少し下、実際の膝の曲がり目に合わせて下げること。
3. **「すね」の線の位置が高すぎる**
   同様にもう少し下、脛の中央〜下寄りに合わせて下げること。
4. **「ゆか」の表現方法を変える**
   この写真は白背景で撮影されているため、床が写真上で視覚的にどこにあるか
   分かりにくい。「ゆか」だけは、他の4つ（ひざ・すね・足首・つま先）と同じ
   「水平の破線＋文字ラベル」のスタイルにする必要はない。
   代わりに、**つま先の位置から前方（画面奥・体の正面方向）に伸びる線**を
   足元に描き足して、「ここに床面がある」と一目で分かるようにすること。
   決して「ゆか」という文字ラベル自体をなくすわけではなく、示し方を変える
   （線を伸ばして床の存在を明示する）という指示。

## ラベルの仕様（変更なし）

- 表示する部位と順番（この5つ、上から）:
  1. ひざ
  2. すね
  3. 足首
  4. つま先
  5. ゆか（表現方法のみ上記4.のとおり変更）
- ひざ・すね・足首・つま先の4つは「対象部位から水平に伸びる破線→文字」のスタイルを維持
- 文字色は黒〜濃灰、破線はベージュ系（現行踏襲）

## やってはいけないこと

- 人物の写真自体の加工・生成（実写を保つ。イラスト化・別人化はNG）
- ラベルの部位・文字・順番を変えること（「ゆか」も消さない。表現方法だけ変える）
- 全体の縦横比・構図を変えること（アプリ側で 2:3 比率を前提に表示している）

- 元画像と同じアスペクト比（900×1350、縦長2:3）のJPEGまたはPNG
