# Pixel実機検証手順書（Android初実機・2026-07作成）

> **これは何**: テスト基準機 Pixel 10a での初回実機検証の手順。ヘッドレステスト（smoke/webkit）では
> 確認できない「実アプリ・実OSの挙動」だけに絞ってある。結果は各項目の☐にチェック+メモを書き、
> 終わったら開発セッションに報告（WORKING_NOTES.mdへ転記する）。
> 対象URL: **https://kyou-no.ogatore.net/**

## 準備（最初に1回）

- ☐ Wi-Fi接続・Playストアで **Chrome を最新に更新**（SIMは不要）
- ☐ **YouTubeアプリ**にログイン（どのアカウントでもOK）
- ☐ **LINEアプリ**にログイン（Keepメモが使えればOK）
- ☐ Chromeで対象URLを一度開いて表示されることを確認

**テスト間のリセット方法**（「まっさらな初回ユーザー」に戻すとき）:
Chrome → サイトの設定 → `kyou-no.ogatore.net` → データを削除。
ホーム画面にアイコンを追加した後なら、アイコン長押し→アンインストールも行う。
※シークレットタブは使わない（sessionStorage/PWAの挙動が変わりテストにならない）。

---

## テストA: YouTubeアプリ内ブラウザの検出と脱出バナー【最重要・実機未確認の核心】

ヘッドレスでは「偽装referrer」でしか確認できていない。**実機のYouTubeアプリがreferrerを
送るかどうか**が今回いちばん知りたいこと（送らない実装なら検出は発火しない。それはそれで正
=配布投稿側の画像手順が頼みの綱、という設計判断の裏付けになる）。

1. ☐ リセットしてから、YouTubeアプリで適当な動画を開き、**自分のコメントとして対象URLを投稿**
   （テスト後に削除でOK。概要欄にURLを書ける自前動画があればそれでも可）
2. ☐ そのコメントのリンクをタップ → YouTubeアプリ内ブラウザで開く
3. ☐ **期待**: 画面上部バナーに
   「**YouTubeの中でひらいてるみたい📺 右上の「⋮」→「Chromeで開く」を押してね（記録を守るため）**」
   が最優先表示される。オンボーディング（ようこそ画面）とホーム追加ポップアップは**出ない**
4. ☐ 実際に **⋮ → Chromeで開く** が存在し、タップでChromeに脱出できる
5. ☐ 結果メモ: バナーが「出た／出ない」→ __________
   （**出ない場合も故障ではなく重要な実測結果**。その場合は手順3の代わりにオンボが出るはず）

## テストB: LINEアプリ内ブラウザの警告と脱出

1. ☐ リセット後、LINEのKeepメモに**素のURL**（パラメータなし）を送信してタップ
2. ☐ **期待**: 「⚠️ここで使うと記録がきえて見えることがあるよ😢」+「やり方を見る」リンクの
   警告バナー（15px・読める大きさか目視確認）
3. ☐ 「やり方を見る」→ 使い方タブの「まもりかた」ガイドに飛び、**iPhone/Android並記ボックス**が読める
4. ☐ 次に **`https://kyou-no.ogatore.net/?openExternalBrowser=1`** をKeepメモから開く →
   **期待**: LINE内ではなく外部Chromeが直接立ち上がる（β配布リンクの形式）

## テストC: Chromeでのホーム画面追加（A2HS）とPWA

1. ☐ リセット後、Chromeで対象URLを開きオンボーディングを完走
2. ☐ ホーム追加ポップアップの種類を記録:
   **ネイティブのインストールダイアログ**（android-prompt）か、**手動メニュー案内**（android-menu）か
   → __________ （Pixel+最新Chromeならandroid-promptのはず）
3. ☐ インストール実行 → ホーム画面のアイコン・名前・起動時のスプラッシュを目視
4. ☐ アイコンから起動 → **アドレスバーなしの単独アプリ表示**（standalone）になっている
5. ☐ 記録を1件つけてから**機内モードにして再起動** → オフラインでも開き、記録が残っている
6. ☐ ブラウザ版とアイコン版で記録が引き継がれているか（同じChromeプロファイルなら同一のはず）

## テストD: 「二度目のチャンス」再提案カード

1. ☐ リセット後、オンボ完走時のホーム追加ポップアップで**「あとで」**を選ぶ
2. ☐ 動画を見たことにして「きょうやった！」で1日目を記録
3. ☐ **期待**: お祝い表示の直下に、ホーム追加の**再提案カード**が出る（「やり方を見る」ボタン付き）
4. ☐ カードを閉じて再読み込み → **二度と出ない**（一度きり制御）

## テストE: 全体ざっと見（15分）

- ☐ かたさチェック完走（同点になりそうな回答=全問3番目あたりを選ぶ→タイプ表示が自然か）
- ☐ 相談室で2〜3往復（キーボードの出入りでレイアウトが崩れないか）
- ☐ 文字サイズ: Android設定でフォントサイズ「最大」にして主要画面が崩れないか
- ☐ ダークモード（端末設定で切替）で一巡
