## 0. まず安心してほしいこと

**何もしなくても、アプリは止まりません。**

- このアプリはサーバーを持っていません。GitHub Pagesという無料の仕組みでファイルを配っているだけです
- 月々の支払い・APIキー・データベース・ビルド作業、ぜんぶ無し。「放置すると止まる」部品がありません
- 利用者の記録は各自のスマホの中にあります。運営側が守るべきユーザーデータは存在しません
- 動画はYouTube側にあるので、アプリの更新が止まっても中身は生き続けます

やることは「①月1回のカタログ更新」と「②たまの投稿」だけ。開発を続けたい場合だけ③以降を読んでください。

## 2. 月1回・30分の定期運用（エンジニア不要）

**目的**: 非公開になった動画をカタログから外し、新しい動画を取り込む。

Macでターミナルを開いて、リポジトリのフォルダで:

```
npm run catalog:update
npm test
git add -A && git commit -m "カタログ更新" && git push
```

- 1行目: YouTubeに問い合わせて公開状況を確認し、`videos.js` を作り直す（要ネット接続・数分かかる）
- 2行目: 93項目の自動チェック。**「PASS」が出なければpushしない**（そのまま何もしなければアプリは壊れません）
- 3行目: 公開。push後、数分でGitHub Pagesに反映されます

※ nodeとpython3が必要です。入っていなければ [nodejs.org](https://nodejs.org) からLTS版を入れるだけ。
※ 忙しければサボってOK。数ヶ月放置しても、非公開動画のリンクが少し混ざるだけで致命傷にはなりません。

## 3. オガトレ通信の投稿（アプリ内のお知らせ）

`obu-feed.js` をテキストエディタで開き、配列の先頭に1件足してpushするだけ。書き方の見本はファイル冒頭のコメントにあります。写真・音声は `assets/obu/` フォルダに置きます。

投稿が止まっても壊れません（最新投稿が古くなるだけ）。

## 4. 壊れたときの応急処置（覚えるのは1つだけ）

**「直前に戻す」が最強の修理です。** アプリは静的ファイルなので、ファイルを前の状態に戻せば必ず直ります。

```
git log --oneline -5        ← 最近の変更履歴を見る
git revert <壊した変更のID>  ← その変更だけを打ち消す
git push                    ← 公開に反映
```

- どのコミットが原因か分からなければ、動いていた頃の日付のコミットIDを探して `git checkout <ID> -- index.html` のようにファイル単位で戻す手もあります
- 判断に迷ったら**何もしないのが正解**。このアプリは触らなければ壊れません

## 5. 次の担い手への引き継ぎ（人間でも別のAIでも）

「バニラJS（フレームワークなし）の静的PWA」なので、普通のWeb制作者なら誰でも触れます。渡すときは、この順に読んでもらってください:

1. **README.md** — 何のアプリで、何ができるか
2. **WORKING_NOTES.md** — どう作られていて、どこでハマるか（開発履歴の正本）
3. **HANDOFF.md** — 壊れやすい箇所・QAの回し方・開発時の約束
4. **AUDIT-MEMO.md / AUDIT-SAFETY-PROPOSALS.md** — 既知の課題と、本人監修待ちの案件

そして最初に伝える約束は3つだけ:

- **変更したら必ず `npm test`（93チェック）が全部PASSしてからpush**。実機で見るなら `npm run smoke` も
- **古いiPhone対応のため `??` と `?.` は書かない**（書くと古い端末で真っ白になります）
- **相談室の文言（soudan-kb.js）は医療安全に関わるので、尾形さん本人の監修なしに変えない**

AIに引き継ぐ場合も同じです。上の4ファイルを読ませて「HANDOFF.mdの約束を守って」と言えば、ClaudeでもCodexでも他のAIでも文脈を復元できます。実際いまも、この前提でAI同士が交代しながら開発しています。

## 6. 最終手段（開発を完全にやめる場合）

それでもアプリは配り続けられます。判断の目安:

- **相談室の監修が続けられない** → 相談室の入口（FABと入口カード）だけ隠せば、中核機能（かたさチェック・きょうの1本・記録）は無傷で残ります
- **カタログ更新を完全にやめる** → 「動画を探す」に非公開動画が混ざっていきますが、リンク切れが増えるだけで事故にはなりません
- **GitHub Pagesが使えなくなる** → リポジトリのファイル一式を任意のレンタルサーバー/Netlify/Cloudflare Pagesに置くだけで移設完了（ビルド不要のため）。手元のcloneがあれば10分の作業です
